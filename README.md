# RALLY

RALLY is a retargetable lifter that translates symbolized x86-64 NASM assembly into semantically rich, optimizable LLVM IR, fundamentally circumventing the error-prone disassembly and heuristic code/data separation that plague traditional raw-binary lifters. By consuming symbolized assembly, such as the formally verified output of FoxDec (https://github.com/ssrg-vt/FoxDec), RALLY treats the source as a first-class input, utilizing a deterministic grammar to structurally preserve position-independent code (PIC) constructs and symbolic references from the outset rather than reconstructing them via fragile guesswork. To bridge the semantic gap between assembly's implicit machine state and LLVM's explicit value semantics, RALLY introduces an integrated pipeline that recovers control flow, reconstructs ABI-compliant function signatures, and explicitly models architectural state through a %State-threaded execution model. This soundness-first approach produces high-fidelity LLVM IR without the heuristic compromises inherent in binary-first lifting, enabling robust static analysis, compiler-grade re-optimization, and seamless cross-architecture porting (e.g., x86-64 to AArch64) for legacy and position-independent software.

## Setup

Run the build script to set up the project:

```bash
./build.sh
```

## Usage

Use `run_pipeline.sh` to run the lifter pipeline on a symbolized NASM x86-64 assembly file. Files generated by FoxDec are preferred.

```bash
./run_pipeline.sh tests/asm/golden.asm
```

## Executing Output

Refer to `run_tests.sh` for examples on how to handle and execute the output LLVM IR generated by the pipeline.
